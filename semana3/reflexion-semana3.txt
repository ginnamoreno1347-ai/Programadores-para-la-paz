echo "Repositorio:
Es el lugar donde se almacenan todos los archivos y el historial de cambios de un proyecto.
Puede estar en tu computador (local) o en GitHub (remoto).

Commit:
Es como una fotografía de los cambios realizados en el repositorio local.
Cada commit guarda un mensaje descriptivo y permite llevar un registro claro.

Push:
Es el comando que envía los commits desde tu repositorio local hacia el repositorio remoto en GitHub.
De esta forma, los demás pueden ver tus cambios y el proyecto se mantiene sincronizado." > conceptos-git.txt

echo "Mantener un registro de cambios con Git es muy importante en los proyectos de software.
Gracias a este historial, los equipos pueden saber qué modificaciones se hicieron, quién las realizó y en qué momento.
Esto aporta transparencia y facilita la colaboración entre varios desarrolladores sin que se pierda trabajo.
Además, el registro permite identificar rápidamente el origen de errores y volver a versiones anteriores si es necesario.
Cada commit funciona como una evidencia clara del avance del proyecto y ayuda a documentar su evolución.
Git también ofrece seguridad al experimentar, ya que se pueden crear ramas para probar nuevas ideas sin afectar la versión principal.
De esta manera, el proyecto se mantiene ordenado y estable a lo largo del tiempo.
Sin un registro de cambios, sería difícil coordinar tareas y garantizar la calidad del software.
Por eso, Git se convierte en una herramienta esencial para el control y la organización técnica.
En conclusión, mantener un historial de cambios asegura que el proyecto pueda crecer de forma responsable y sostenible." > reflexion-semana3.txt
